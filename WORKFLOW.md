## 1️⃣ 每日内容生产流程（30分钟）

### 步骤 1: 自动采集选题（07:00）

**自动执行**（定时任务）:
```bash
# 系统自动运行
python3 scheduler.py
```

**或手动触发**:
1. 打开 http://localhost:8501
2. 进入「📊 选题采集」页面
3. 点击「🔄 开始采集选题」

**系统会做什么**:
```
采集数据源
├── 知乎热榜（前10条）
├── Google Trends（前5条）
├── 澎湃新闻教育频道（前5条）
└── RSS 订阅（科学网等）

↓

Claude AI 智能打分
├── 相关度（0-10分）
├── 时效性（0-10分）
├── 互动潜力（0-10分）
└── 独特性（0-10分）

↓

生成结果
├── 总分排序
├── AI 评分理由
└── 建议切入角度
```

**输出示例**:

### 步骤 3: 生成 5 种格式内容（07:10-07:15）

**操作**:
1. 进入「✍️ 内容生成」页面
2. 确认选题和观点
3. 点击「🚀 生成所有格式内容」

**系统处理流程**:
```
1. 搜索 OpenClaw 知识库
   └── 查找相关的历史观点和案例

2. 调用 Claude API 生成内容
   ├── 🎬 视频号文案（800-1200字，3-5分钟口播）
   ├── 📰 公众号文章（1800-2200字，深度分析）

   └── 📚 书稿素材（700-900字，学术深度）

3. 自动分类书稿素材
   └── Claude 分析内容，归类到对应章节
```

## 2️⃣ 知识库注入流程

### 方式 1: 收件箱监听（自动）

**设置**:
1. 打开「📚 知识库管理」页面
2. 点击「🚀 启动收件箱监听」

**使用**:
```
1. 将文件拖入收件箱
   ~/ai-content-factory/data/inbox/

2. 系统自动处理
   ├── 等待 2 秒（确保文件完整）
   ├── 读取文件内容
   ├── 检测编码（支持多种编码）
   ├── 提取文本
   ├── 分块处理（2000字/块）
   ├── Claude 分类到书稿章节
   ├── 写入 OpenClaw 知识库
   └── 写入 Notion 数据库

3. 处理完成后移动到
   ~/ai-content-factory/data/imported/2026-02-19/
```

**支持格式**:
- .txt（自动检测编码）
- .md（Markdown）
- .docx（Word 文档）
- .pdf（PDF 文档）

**提示词识别**:
```
如果文件包含"提示词"/"prompt"/"模板"关键词
→ 系统会提取 prompt 片段
→ 返回给你确认
→ 不会自动写入（需要你手动确认）
```

### 步骤 3: Prompt 优化

**操作**:
1. 切换到「🎯 提示词优化」Tab
2. 选择平台（视频号/公众号/小红书等）
3. 点击「🤖 生成优化建议」

**AI 分析**:
```
系统会:
1. 读取该平台所有内容的效果数据
2. 分析高分内容的共同特征
3. 对比低分内容找出问题
4. 生成 Prompt 优化建议

输出示例:
---
# 视频号文案优化建议

## 数据分析
- 高分内容（≥4星）平均阅读量: 6800
- 低分内容（<3星）平均阅读量: 1200
- 差距原因: 开头吸引力不足

## 当前问题
1. 开头不够吸引人，前3秒流失率高
2. 缺少情感共鸣点
3. CTA 不够明确

## 优化方向
1. **开头优化**: 用反常识问题或冲突场景
2. **情感共鸣**: 增加真实案例和场景描述
3. **节奏控制**: 每30秒一个小高潮
4. **CTA 优化**: 明确引导互动

## Prompt 调整建议
在开头部分增加:
"用一个让家长意外的问题或场景开场，
前3秒必须抓住注意力"

在主体部分增加:
"讲述一个真实的家长案例，
让观众产生'这说的就是我'的共鸣"

在结尾增加:
"用一个开放式问题引导评论，
例如'你家孩子遇到过这种情况吗？'"
---

点击「💾 下载优化建议」
→ 保存为 Markdown 文件
```
